# Right Side Navigation

## Introduction

GitBook automatically creates right-side navigation using headings.

## Why It Is Important

Right-side navigation helps users quickly move between sections.

## Best Practices

### Use Proper Headings

Use markdown headings consistently.

### Keep Content Structured

Divide content into sections and sub-sections.

### Improve Readability

Use bullet points and short paragraphs.

## Example Structure

A documentation page should contain:
- Main headings
- Sub-headings
- Lists
- Organized content

## Conclusion

Right-side navigation improves documentation usability.
